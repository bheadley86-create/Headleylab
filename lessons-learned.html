Overview

I built Headley Lab to turn certification study into practical experience. Instead of only memorizing commands and definitions, I used Windows 11 Pro, Hyper-V, Windows Server, Ubuntu Linux, and my home network to practice administration, security, troubleshooting, recovery, and documentation.

My normal workflow became Plan → Do → Verify → Explain. I created checkpoints before risky VM changes, changed one item at a time, verified the result, and documented what worked or failed.

Virtualization and Windows Administration

  • Configured Hyper-V on Windows 11 Pro.
  • Created Windows 11, Windows Server, and Ubuntu virtual machines.
  • Used checkpoints before configuration changes.
  • Practiced Task Manager, Services, Event Viewer, Task Scheduler, Disk Management, Defender, Firewall, Recovery Environment, local users, permissions, and PowerShell.
  • Created scheduled maintenance tasks and enabled PowerShell transcription.
  • Applied least privilege by removing unnecessary administrator access.

Active Directory, DNS, and Domain Administration

  • Installed Active Directory Domain Services and DNS.
  • Created the batcave.local domain.
  • Configured the domain controller at 10.10.10.10 and a client at 10.10.10.20.
  • Joined Windows clients to the domain and used domain accounts.
  • Worked with DNS forward and reverse lookup zones, account permissions, password policy, and Remote Desktop.
  • Learned how DNS affects authentication, Group Policy, and network resources.

Networking and Connectivity Troubleshooting

  • Used ipconfig, ping, nslookup, tracert, pathping, netstat, and Test-NetConnection.
  • Diagnosed an APIPA problem and corrected static IPv4 and DNS settings.
  • Learned to troubleshoot in layers: configuration, connectivity, routing, DNS, ports, services, and applications.
  • Configured DHCP reservations for important devices.
  • Worked with a Wi-Fi 7 router, Deco access points, and an Omada managed switch.
  • Used separate primary, guest, and IoT wireless networks.
  • Used a Wi-Fi analyzer to evaluate signal strength and wireless conditions before selecting access point placement.
  • Terminated an Ethernet cable with RJ45 connectors using the correct wiring layout.
  • Ran the cable to the access point and established a wired backhaul connection to the network.

Remote Administration

  • Used Remote Desktop and verified account and firewall requirements.
  • Configured WinRM and PowerShell Remoting.
  • Installed and used OpenSSH on Ubuntu.
  • Connected to Ubuntu through SSH from Windows Terminal.
  • Verified reachability, listening ports, firewall rules, services, and authorization when troubleshooting remote access.

Linux Administration and Scripting

  • Practiced navigation, file management, permissions, package management, networking, processes, and services.
  • Used ls, cd, mkdir, cp, mv, rm, chmod, ip, dig, nslookup, curl, ps, top, systemctl, and apt.
  • Created Bash scripts using variables, conditionals, loops, command substitution, and redirection.
  • Created and executed a storage-check script.

Security, Malware Response, and Recovery

  • Practiced Defender scans, Firewall review, Controlled Folder Access testing, and protection-history review.
  • Performed a simulated CompTIA malware-removal scenario.
  • Isolated the system, disabled System Restore, scanned and remediated, updated protection, scheduled scans, re-enabled protection, and documented the result.
  • Decided that a reset, reimage, or clean installation is appropriate when a system can no longer be trusted.
  • Used Bitwarden, MFA, and a YubiKey for account security.

Storage, Backup, and Recovery

  • Worked with backup locations, restore points, recovery options, virtual disks, and system images.
  • Practiced initializing, partitioning, formatting, labeling, and verifying disks.
  • Used Windows Recovery Environment tools.
  • Learned that System Restore is not a full file backup.
  • Planned a Raspberry Pi 5 NAS project as a future deployment.

Troubleshooting Results and Failures

  • Fixed an APIPA and DNS problem that blocked domain communication.
  • Confirmed that a system can have IP connectivity while DNS or domain services are still broken.
  • Learned that the HOSTS file can override DNS.
  • Practiced NTFS and share permissions.
  • Documented unresolved Windows Sandbox errors instead of hiding the failed result.
  • Separated licensing problems from operating-system and network problems.

Main Lessons Learned

  1. DNS is foundational.
  2. Troubleshoot one layer at a time.
  3. Create a checkpoint before risky changes.
  4. Verification matters more than clicking.
  5. Least privilege reduces risk.
  6. Security includes deciding whether a system is still trustworthy.
  7. A failed lab can still produce useful evidence.
  8. Documentation turns practice into experience.
  9. Wireless placement should be based on measured signal conditions, and wired backhaul can improve access point reliability and performance.
